遇到的问题以及解决方案

Jackson 处理大小写的问题

原因：前端传JSON的key为vImage，后端bean中的属性名也为vImage，调接口接收时却没有收到vImage字段的值，报错显示该值为null，其他字段正常接收到。

解析：

一般情况下，前后端字段名一致时SpringMVC可以正确匹配上，但是某些特殊大小写的情况时却会出现不匹配的现象。后端SpirngMVC序列化JSON是交给Jackson处理，那么Jackson的机制究竟是怎么样的？

在Bean中，Jackson会获取属性对应的get方法方法名，比如getXxx，然后进行将get进行截断，变成Xxx，最后将其小写，变成xxx。因此Jackson序列化的key定义与field名称无关，反而和其get方法名称有关；

其次，jackson只会将连续的大写转换成小写。如果中间断了，之后的大写字符也不会处理了。

并且Jackson是从开头检测的，如果开头就是小写，那么之后的大写字符也不会处理了。

例如：

bean如下：

@Data
@NoArgsConstructor
@AllArgsConstructor
public class AddGoodsDTO {
    private String vImage;
}

字段vImage的get方法为public String getVImage()，这一般由IDE自动生成的。

Jackson解析时，会自动将get截断，取得VImage，在将第一个大写V转成小写v，成为vImage。当然这是我们以为会是这样。

实际上当两个大写字母挨在一起时，他会继续将大写字母转成小写。因此Jackson在处理VImage时，实际上转成了vimage。所以前端的JSON的key为vImage时自然找不到了。

解决方案：

  1. 在field和其对应的get方法上加上对应的标签(@JsonProperty和@JsonIgnore)，然后jackson在序列化的时候就会以你的field名称为key。

    @Data
@NoArgsConstructor
@AllArgsConstructor
public class AddGoodsDTO {
    @JsonProperty
    private String vImage;
}

@JsonIgnore
public String getVImage(){
    return this.vImage;
}

  2. 使用其他的序列化工具。

@JsonFormat注解时，LocalDateTime反序列化失败

原因：@JsonFormat只支持Date类型，对于jdk8新出的时间API不支持

解决方案：

  1. 将LocalTime类型改成Date类型。

  2. 自定义Jackson的配置。

    //Jackson配置类
@Configuration
public class JacksonConfig {

    @Bean
    public ObjectMapper objectMapper() {
        ObjectMapper objectMapper = new ObjectMapper();
        
        // 创建一个JavaTimeModule对象来处理Java 8中的日期和时间类型
        JavaTimeModule javaTimeModule = new JavaTimeModule();
        
        // 添加LocalDateTime的序列化器，将其格式化成指定的"yyyy-MM-dd HH:mm:ss"格式
        javaTimeModule.addSerializer(LocalDateTime.class, new LocalDateTimeSerializer(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss")));
        
        // 添加LocalDate的序列化器，将其格式化成指定的"yyyy-MM-dd"格式
        javaTimeModule.addSerializer(LocalDate.class, new LocalDateSerializer(DateTimeFormatter.ofPattern("yyyy-MM-dd")));
        
        // 添加LocalTime的序列化器，将其格式化成指定的"HH:mm:ss"格式
        javaTimeModule.addSerializer(LocalTime.class, new LocalTimeSerializer(DateTimeFormatter.ofPattern("HH:mm:ss")));
        
        // 添加LocalDateTime的反序列化器，将指定格式的字符串转换为LocalDateTime对象
        javaTimeModule.addDeserializer(LocalDateTime.class, new LocalDateTimeDeserializer(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss")));
        
        // 添加LocalDate的反序列化器，将指定格式的字符串转换为LocalDate对象
        javaTimeModule.addDeserializer(LocalDate.class, new LocalDateDeserializer(DateTimeFormatter.ofPattern("yyyy-MM-dd")));
        
        // 添加LocalTime的反序列化器，将指定格式的字符串转换为LocalTime对象
        javaTimeModule.addDeserializer(LocalTime.class, new LocalTimeDeserializer(DateTimeFormatter.ofPattern("HH:mm:ss")));
        
        // 注册JavaTimeModule以支持Java 8中的日期和时间类型
        objectMapper.registerModule(javaTimeModule);
        
        return objectMapper;
    }
}

如何根据Token知晓操作人是谁？该如何保存和使用？

原因：因为已经使用jwt令牌技术，因此每一次操作实际上可以根据Token来知晓操作人的信息，但是如何将该操作人信息保存并且全局可以使用？

解决方案：使用ThreadLocal。

ThreadLocal并不是一个Thread，而是Thread的局部变量。

客户端发起的每一次请求，Tomcat都会单独开辟一条线程，ThreadLocal为每个线程提供单独一份存储空间，具有线程隔离的效果，只有在线程内才能获取到对应的值，线程外则不能访问。因此可以将信息放入ThreadLocal中。
